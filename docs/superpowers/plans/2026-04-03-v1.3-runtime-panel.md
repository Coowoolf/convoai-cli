# v1.3 Runtime Control Panel — Implementation Plan (Part 2 of 2)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task.

**Goal:** After Agent starts, show a live status panel with keyboard shortcuts to hot-update ASR/LLM/TTS/VAD parameters without restarting.

**Architecture:** New `src/commands/agent/panel.ts` module provides `runPanel(api, agentId, channel, lang)`. Quickstart and agent join call it after agent starts. Uses raw stdin for single-key detection, inquirer for sub-menus, and the Agora update API for hot changes.

**Tech Stack:** Node.js readline (raw mode), inquirer (sub-menus), Agora update API, existing gradient/i18n modules.

### Task 1: Create the runtime control panel

**Files:**
- Create: `src/commands/agent/panel.ts`

The panel has two modes:
1. **Main panel** — status display + key listener
2. **Sub-menu** — inquirer prompt for parameter changes

Main panel renders:
```
  ⚡ Agent 运行中 · A42AX... · Channel: my-room
  ─────────────────────────────────────────

  🎙 ARES (zh-CN)        🧠 qwen-plus
  🔊 MiniMax             ⏱ VAD 1000ms

  对话: 3 轮 · 平均延迟 1.2s

  ─────────────────────────────────────────
  [a] ASR  [l] LLM  [t] TTS  [v] VAD  [h] 历史  [q] 退出
```

Auto-refreshes every 3 seconds (polls history + turns for stats).

Key handler:
- `a` → ASR sub-menu (switch language, switch vendor)
- `l` → LLM sub-menu (switch model, switch vendor, temperature, max_tokens, system prompt)
- `t` → TTS sub-menu (switch vendor, speed, volume)
- `v` → VAD sub-menu (silence_duration, interrupt_duration, interrupt_mode)
- `h` → show conversation history
- `q` → exit (stop agent, show session report)

Sub-menus use inquirer prompts. After each change:
- If update API supports it (LLM model/temperature/max_tokens/system_messages) → call `api.update(agentId, {...})`
- If requires restart (vendor change, VAD) → prompt "需要重启 Agent，当前对话将断开。是否继续？(Y/n)", then stop + start

Export: `runPanel(opts: { api: AgentAPI, agentId: string, channel: string, lang: 'cn' | 'global', config: ProfileConfig }): Promise<void>`

i18n: use `getStrings(lang)` for all labels. Add panel-specific strings to i18n.ts.
